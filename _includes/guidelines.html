Processes & guidelines

Contribution guidelines

YaST is an open source project and as such it welcomes all kinds of contributions. Either if you want to report a bug or if you want to contribute some code, make sure you read the Yast contribution guidelines first.

When contributing code, it's important to observe some conventions. Yast follows the SUSE's Ruby style guide for the source code and its own UI style guide.

From the repository to the distribution

Yast honours the "release early, release often" principle with a continuous integration infrastructure. The contribution process does not end when the code enters the repository. The full development workflow also includes the automatic creation of packages and integration in the distributions and its described in depth in this page of the openSUSE wiki.
