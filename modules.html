What can you configure with YaST?

YaST offers a vast catalog of modules to configure almost every single piece of a Linux system.

yast-add-on
Allows to add/remove add-ons for base system.
yast-add-on-creator
This module can create custom add-on products from provided RPM packages.
yast-apparmor
Apparmor profile management.
yast-auth-client
Configuration of various authentication methods for system like kerberos, ldap or sssd.
yast-auth-server
Configuration of server providing various authentication methods like kerberos, ldap or sssd.
yast-bootloader
Bootloader installation and configuration.
yast-ca-management
Certificate authority management.
yast-cio
Management of blacklisting channels for Input/Output for s390x.
yast-cluster
Management of settings for high availability clusters.
yast-country
Configuration of localization specific settings like language, keyboard layout, timezone, etc.
yast-crowbar
Configuration of Crowbar, the orchestation tool used by SUSE Cloud.
yast-dbus-server
YaST DBus service providing DBus access to YaST components.
yast-dhcp-server
Configuration of DHCP server, including LDAP-related settings.
yast-dns-server
Comprehensive configuration of BIND DNS server.
yast-docker
Management of Docker daemon, images, and containers.
yast-drbd
Configuration of Distributed Replicated Block Devices, a distributed storage system frequently used on high availability (HA) clusters.
yast-fcoe-client
Settings to connect to a Fibre Channel over Ethernet (FCoE) network.
yast-firewall
Management of SUSE Firewall.
yast-firstboot
Allows system administrators to setup a wizard to execute on the first run of a system. Useful for image-based deployments.
yast-fonts
Management of fontconfig settings to control font appearance, including font rendering algorithm and font family preference.
yast-ftp-server
Configuration of Pure-FTPd and vsftpd.
yast-geo-cluster
Management of Geo Clustering for high availability.
yast-http-server
Comprehensive configuration of Apache2.
yast-inetd
Configuration of inetd and xinetd daemons.

Client side of network information services (NIS, YP), such as user and group information, and hostnames.
yast-nis-server
Server configuration for network information services (NIS, YP).
yast-ntp-client
Time synchronization over network (NTP).
yast-online-update
Updates your SUSE with the latest patches from online repositories.
yast-opensuse-mirror
Configures mirroring of openSUSE repositories locally.
yast-pam

yast-rear
Configure backup using Relax and Recover (ReaR).
yast-registration
Registeres the system to the SUSE Customer Center (SCC) or Subscription Management Tool (SMT) server to receive security updates, extra software and customer support (SUSE Linux Enterprise only).
yast-reipl
IBM zSeries specific configuration (loading IPL (initial program loader) from a running system.
yast-s390
IBM zSeries specific configuration (DASD disks, ZFCP disks, IUCV terminal, ...).
yast-samba-client
Client configuration for using file and printing services over SMB/CIFS networking protocol provided by Windows or Samba server.
yast-samba-server
Server side of a file and printing services using Samba.
yast-scanner
Configures local (SCSI, USB) or network scanners.
yast-security
Security settings (password and login settings, kernel options...)
yast-services-manager
Configures system services started at boot (systemd services).
yast-slp-server
Configures the OpenSLP server.
yast-snapper
Configures snapper (the Btrfs snapshotting tool), can browse snapshots.
yast-sound
Configures sound cards (usually needed just in some special cases, autoconfiguration should work out-of-box).
yast-squid
Configures Squid proxy servise.
yast-storage
Disk and partition management.
yast-sudo
Configures the sudo tool (to execute commands as another user or as root).
yast-sysconfig
Editor for /etc/sysconfig files.
yast-tftp-server
Configures TFTP server (used for booting over network).
yast-tune
Hardware detection and expert system tuning.
yast-users
User management module - manages users and groups.
yast-vm
Installs tools for using XEN or KVM virtualiztion.
